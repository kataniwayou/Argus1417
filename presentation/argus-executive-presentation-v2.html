Core Engine: CentralTimer

The heartbeat that drives all monitoring operations

How It Works

1

Tick Event Fires

Every 10 seconds (configurable), CentralTimer emits a tick event

2

Callbacks Execute

All registered callbacks run: K8s polling, heartbeat checks, alert processing

3

Liveness Stamp

Each callback must stamp the LivenessVector on completion

4

Health Check

Callbacks that fail to stamp are marked unhealthy

Registered Callbacks

K8sLayerCallback HeartbeatExpiryCallback NocQueueCallback SnapshotCallback NasHeartbeatCallback

Code Example

// CentralTimer tick handler private async Task OnTickAsync() { foreach (var callback in _callbacks) { try { await callback.ExecuteAsync(); _livenessVector.Stamp(callback.Name); } catch (Exception ex) { _metrics.RecordCallbackError(callback.Name); _logger.LogError(ex, "Callback failed"); } } _metrics.IncrementTickCount(); }

The Problem with Traditional Monitoring

Traditional monitors can fail silently. A hung thread, a deadlock, or an exception loop can leave your monitoring blind without any indication.

LivenessVector Solution

    Callback Registration

    Each callback registers with LivenessVector on startup

  • ‚úì

    Stamp on Completion

    Callbacks must call Stamp() after successful execution

  • ‚è∞

    Expiry Detection

    Callbacks that miss stamps are marked unhealthy

  • üìä

    Health Metrics

    Exposed via OTEL for Grafana alerting

LivenessVector State

// LivenessVector tracks callback health public class LivenessVector { private readonly ConcurrentDictionary<string, CallbackState> _states; public void Register(string callbackName) { _states[callbackName] = new CallbackState { LastStamp = DateTime.UtcNow, IsHealthy = true }; } public void Stamp(string callbackName) { _states[callbackName].LastStamp = DateTime.UtcNow; _states[callbackName].IsHealthy = true; } public bool IsHealthy() => _states.Values.All(s => s.IsHealthy); }

High Availability: Leader Election

Kubernetes Lease-based coordination for zero-downtime operations

How It Works

Key Behaviors

üîÑ All Pods Run Identical Logic

Every pod executes the same callbacks, maintains the same AlertVector state. Only NOC sending is gated by leadership.

‚ö° Automatic Failover

If the leader pod dies, another pod acquires the lease within seconds. No manual intervention required.

üõ°Ô∏è No Duplicate Alerts

Only the leader sends to NOC. Followers process alerts but suppress NOC output, preventing duplicates.
