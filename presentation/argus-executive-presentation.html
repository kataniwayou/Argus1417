Argus Components

Argus Component Details

CENTRAL TIMER
  • 1-second tick engine, maintains tick counter + timestamp
  • Registers callbacks with configurable intervals
  • Grace period skips "grace-aware" callbacks
  • Generates correlation ID per tick for tracing
LIVENESS VECTOR
  • Tracks callback health via tick-based stamping
  • Tolerance: 2√ó expected interval
  • Exception = no stamp = detected as unhealthy
COORDINATOR
  • Orchestrates K8sPolling & NocSnapshot callbacks
  • Filters Prometheus alerts by platform="argus"
  • Routes watchdog alerts, generates execution IDs
RESTART TRACKER
  • Sliding window detects pod restart storms
  • Reports stable during grace period
  • Used by K8s Layer for pod health checks
METRICS
  • OpenTelemetry-based: ingestion, lifecycle, NOC, health
  • Exported via OTel Collector
ALERTS VECTOR
  • In-memory, priority-ordered, fingerprint-keyed
  • CREATE/CANCEL status with TTL-based cleanup
  • Updated asynchronously by all alert sources
SUPPRESSION CACHE
  • Prevents duplicate NOC via fingerprint:status
  • Configurable time windows (e.g., "2m", "5m")
  • Failed NOC = unmarked for immediate retry
NOC DECISION QUEUE
  • Async queue with background consumer
  • Leader sends HTTP, follower verifies
  • Removes CANCEL alerts from vector on success
NOC HEALTH SERVICE
  • Circuit breaker: tracks consecutive NOC failures
  • Trips at threshold ‚Üí stops file heartbeat
  • Resets on any successful NOC call
